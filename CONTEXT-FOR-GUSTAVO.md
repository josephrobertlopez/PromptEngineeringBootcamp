# AI Coaches Enablement — Design Principles

## Principle 1: Tool-Agnostic
All content must work identically in ChatGPT, Claude, Copilot, and Windsurf. Teach patterns, not products. Tools change. Patterns don't.

## Principle 2: Felt Before Named
People learn by doing, then naming. Exercise first, explanation second. Never define a concept before the learner has experienced it.

## Principle 3: Three Intuitions (Foundation Layer)
Every AI interaction is one operation: assembling context. Three intuitions capture this:
- **Context is everything.** More relevant context = better output. Always.
- **Structure gets rewarded.** Organized input → organized output. The model was trained for this.
- **You are the retrieval system.** Every tool/skill/workflow just automates: retrieve context → assemble it → generate output.

These three intuitions explain Skills, Workflows, Prompt Files, and RAG without needing to teach those terms directly.

## Principle 4: Starter Kit → Skill → Build Your Own
Three stages per role path:
1. **Starter kit** — paste-and-go templates, first real output in 5 minutes
2. **Context Prompt** — interrogation-driven skill that personalizes everything to their current project
3. **Build your own** — participant creates a reusable tool for a task they repeat weekly

## Principle 5: Minimum Viable Path
Shorter path to first real output = higher completion. Core curriculum only at launch. Reference material released as needed during pilot.
