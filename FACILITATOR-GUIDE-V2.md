# SESSION: Prompt Engineering Bootcamp v2 (90 minutes)

## **PRE-SESSION SETUP (30 minutes before)**

### **Materials Ready Checklist:**
- [ ] **LIVE-SESSION-V2 slides** → Loaded and tested (slides 1-21 displaying)
- [ ] **COMMON-KNOWLEDGE.md** → Sent to participants 3-5 days prior
- [ ] **Quick reference cards** → quick-reference-cards.md printed (one per participant)
- [ ] **Skill files ready to distribute:**
  - [ ] dev-second-brain/SKILL.md
  - [ ] po-second-brain/SKILL.md
  - [ ] dl-second-brain/SKILL.md
  - [ ] tl-second-brain/SKILL.md
  - [ ] make-skills/SKILL.md
- [ ] **AI tool access** → ChatGPT/Claude/Copilot ready (any works)
- [ ] **GitHub repo link** → Ready to share or email
- [ ] **Backup printed prereqs** → 2-3 copies for non-completers

### **Room Setup**
- Projector with LIVE-SESSION-V2 slides loaded
- Each participant: laptop, AI tool access
- Track groups seated together (optional: pre-assign tracks)
- Distribution method decided: GitHub link, shared drive, or email

### **Track Assignment**
- Pre-assign tracks based on role (recommended) OR self-select at session start
- Ideal group size: 3-6 per track
- If single person in track, pair with closest track (DL↔TL, Dev↔TL, PO↔DL)

### **Prereq Verification**
- Send COMMON-KNOWLEDGE.md 3-5 days before session
- Send reminder 1 day before: "Complete the 15-minute prereq before tomorrow's session"
- Have printed copies for anyone who didn't complete (they read during Part 1)


### **Common Issues & Responses**:

**Issue**: "The skill is asking too many questions"
**Response**: "Answer them. That's the retrieval happening. It's gathering context you wouldn't have included."

**Issue**: "My AI tool cut off the skill"
**Response**: "Paste it in smaller chunks, or use Claude which handles long inputs better."

**Issue**: "I'm not sure how to answer question X"
**Response**: "Say 'I don't know' or 'skip this one.' The skill adapts."

### **Time Calls**:
```
35:00 - "More than halfway through"
40:00 - "10 minutes left"
42:00 - "5 minutes left"
42:30 - "If skill is still asking questions, tell it to generate with what it has"
47:00 - Transition: "Compare your outputs"
```

### **Facilitator Role**:
Walk the room. Help with logistics (loading the skill, pasting into AI tool). Do NOT help with content — the skill handles that.

## **Common Failure Modes**

### **1. Re-teaching the Prereq**
**Problem**: Spends 5+ minutes explaining Intuition 1-3
**Impact**: Feels like remedial education, kills momentum
**Fix**: Don't. If someone didn't complete it, they pair with someone who did. You have 90 minutes, not 120.

### **2. Over-explaining Patterns**
**Problem**: Spends 10 minutes explaining RAG/LLM/RLHF at minute 75
**Impact**: The reveal feels academic, not felt
**Fix**: The prereq installed the intuitions. Names are labels, not lessons. Read slides. Pause.

### **3. Helping Too Much During Freestyle**
**Problem**: Suggests patterns, gives examples during 17-27 minute exercise
**Impact**: The felt contrast is lost — they don't feel the difference anymore
**Fix**: Walk the room. Don't help. Let them struggle.

### **4. Rushing the Reveal**
**Problem**: Reads slides 16-17 in 2 minutes, moves on
**Impact**: Misses the payoff moment
**Fix**: This is the highest-ROI 5 minutes of the session. Read slowly. Pause. Let it land.

### **5. Asking for Questions at the End**
**Problem**: "Any questions before we wrap?"
**Impact**: Kills momentum, extends by 5-10 minutes, awkward silence
**Fix**: End with "Go build." Stay available after. Don't ask.

- Send skill files to breakout room chat before exercise starts

### **Shared Output** (47-52 min comparison)
- Use Google Doc or shared screen for 1-2 outputs

### **Capstone** (55-75 min)
- Can be individual in main room or breakout
- Share button lets people show skill output to larger group

### **Screen Sharing**
- Pre-test all facilitators' screen share
- Have backup: email skill files in case chat uploads fail
