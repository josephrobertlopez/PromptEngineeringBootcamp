KEY FEATURES
================================================================================

Three Bootcamp Intuitions Embedded:
  1. Context is everything — 25-question interrogation phase
  2. Structure gets rewarded — All outputs follow templates (stories, backlogs, reports)
  3. You are the retrieval system — Skill assembles project knowledge automatically

Spec-Kit Methodology:
  Knowledge → Specification → Plan → Execution
  - Used throughout the skill
  - Demonstrated in working examples
  - Prevents missing critical context

Four Usage Scenarios:
  A) Requirements Capture → User stories with Given/When/Then
  B) Sprint Planning → Sprint backlogs with capacity, dependencies, burn-down
  C) Stakeholder Communication → Executive reports with risks, metrics, asks
  D) Roadmap Planning → Value/effort/risk prioritization across quarters

Interrogation-Driven Workflow:
  Phase 1: Business Context (6-8 questions)
  Phase 2: Scope & Requirements (5-7 questions)
  Phase 3: Team & Capacity (3-4 questions)
  Phase 4: Acceptance & Validation (3-4 questions)
  Phase 5: Knowledge & Decisions (2-3 questions)

Output Formats (All Structured):
  • User Stories with acceptance criteria (Given/When/Then format)
  • Sprint Backlogs with capacity planning
  • Stakeholder Reports with executive summaries
  • Prioritized Roadmaps with value/effort/risk scoring

DESIGN PATTERNS DEMONSTRATED
================================================================================

Pattern 1: Interrogation Before Generation
  - Never generate without understanding context
  - 25 questions surface: constraints, dependencies, team capacity, risks
  - Prevents "missing requirement" after-the-fact surprises

Pattern 2: Given/When/Then Over Vagueness
  - Every acceptance criterion is testable
  - Given [precondition] / When [action] / Then [outcome]
  - Makes "done" unambiguous

Pattern 3: Structured Artifacts Over Prose
  - User story template (not narrative)
  - Sprint backlog table (not bullet list)
  - Report with sections (not long email)
  - Roadmap with scores (not wishlist)

Pattern 4: Spec-Kit Methodology
  - Knowledge: What did interrogation reveal?
  - Specification: What are we building? (stories)
  - Plan: How will we build it? (sprint, timeline)
  - Execution: Who does what? (ownership, dependencies)
